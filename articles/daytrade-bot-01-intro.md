## なぜボットを作ろうと思ったか

正直に言うと、最初は「AIに任せれば勝手に稼いでくれる」くらいの気持ちでした。

でも調べれば調べるほど、**本当に難しい**ということがわかってきました。

2026年5月時点の最新研究（arXiv 2505.07078）によれば、LLMを使ったトレードエージェントの多くは、単純なバイアンドホールドにすら勝てないという結果が出ています。

じゃあなぜ作るのか。

理由は3つです。

1. **感情を排除できる** 人間は損切りできない。ボットは機械的にルールを守る
2. **24時間監視できる** 仕事中も、寝ている間も動き続ける
3. **データが蓄積される** 実際の取引記録からAIが毎日自動で改善する

「AIが予測して稼ぐ」ではなく、「AIがルールを守って改善し続ける」という設計思想です。

## 使った技術スタック

```
言語:       Python 3.11
価格データ: yfinance（無料・15分遅延）
           kabu STATION API（無料・リアルタイム）
取引API:   kabu STATION API（auカブコム証券）
DB:        SQLite（取引ログ）
通知:      LINE Messaging API（無料）
画面:      Flask + Chart.js（ダッシュボード）
AI:        Anthropic Claude API（毎日の反省会）
OS:        Windows 11（タスクスケジューラーで自動起動）
```

全部合わせてほぼ無料で動きます。  
唯一コストがかかるのはClaude API（毎日の反省会で数円〜数十円程度）。

## 1日の自動動作フロー

PCの電源を入れるだけで、あとは全部自動です。

| 時刻 | 自動で起きること |
|------|----------------|
| 08:45 | ボット自動起動（タスクスケジューラー） |
| 08:50 | 238銘柄をスクリーニング → LINEに銘柄リスト送信 |
| 09:00〜 | 60秒ごとにシグナル判定・売買実行 |
| 13:30 | 新規エントリー禁止（後場後半は勝率が低いため） |
| 15:00 | 全ポジション強制決済 |
| 15:30 | 日次レポートをLINEに送信 |
| 16:00 | AI（Claude）が取引を分析してパラメータを自動改善 |

## リスク管理の考え方

ボットで一番怖いのは「暴走」です。

なので7つの層でエントリーを禁止するルールを設けています。

1. ポジション数が上限（資金に応じて1〜5）に達している
2. 13:30以降（後場後半は文献・バックテスト両方で劣勢が確認済み）
3. 本日の損益が日次損失上限（資金×2%）を超えた
4. **3連敗した（その日の取引を全停止）**
5. **日経VI（恐怖指数）≥ 25（市場が荒れている）**
6. その銘柄のR:R比（利益÷損失）< 2.0
7. シグナルスコアが閾値未満

特に4・5・6は、2026年の最新論文を読んで追加した機能です。

## バックテストの結果（先に言います）

5分足データ（直近60日・48銘柄）でのシミュレーション結果です。

| 指標 | 結果 |
|------|------|
| 総取引数 | 1,322回（1日平均22回） |
| 勝率 | 52.1% |
| ペイオフレシオ | 1.63倍 |
| 累積損益（シミュレーション） | +2,019,956円 |

ただしこれはシミュレーションです。実際の取引ではスリッページや流動性の問題が出ます。過信は禁物。
